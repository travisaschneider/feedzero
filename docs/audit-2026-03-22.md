# FeedZero Codebase Audit — 2026-03-22

Full audit across security, performance, simplicity, auditability, and reliability.

**Overall verdict:** Strong foundation with real gaps. Clean architecture, rigorous testing (906 tests, 90% coverage), well-thought-out privacy model. Critical issues remain in SSRF protection, KDF parameters, performance at scale, and operational readiness.

---

## Critical (Fix before production)

### 1. SSRF Bypass — IPv6-Mapped & Alternate IP Notation

- [x] Fix (2026-03-22)

**Location:** `src/core/proxy/validate-url.ts:41-48`

The validator blocks `127.0.0.1` but not `::ffff:127.0.0.1`, `0x7f.0.0.1`, `0177.0.0.1`, or `2130706433` (integer form). Complete bypass of all private IP protections.

**Fix:** Parse and validate IPv6-mapped addresses. Reject octal, hex, and integer IP notation. Normalize hostnames before checking against blocklists.

---

### 2. PBKDF2 Iterations Too Low

- [x] Fix (2026-03-22)

**Location:** `src/utils/constants.ts:9` — currently 100,000 iterations

OWASP 2023 minimum is 600,000 for SHA-256. With a 4-word passphrase (~51.7 bits entropy), a single RTX 4090 makes brute-force feasible. This is the foundation of the entire encryption model.

**Fix:** Increase to 600,000. Requires migration path for existing users (re-derive keys on next open).

---

### 3. DEFAULT_PASSPHRASE for All Local Users

- [x] Fix (2026-03-22)

**Location:** `src/utils/constants.ts:15` — `"feedzero-default-key"`
**Used at:** `src/stores/app-store.ts:73`

Every local-only user shares the same encryption key. Zero protection against data exfiltration from a stolen laptop or browser extension.

**Fix:** Generate a random per-user passphrase on first run and store the derived keys. Remove `DEFAULT_PASSPHRASE` constant.

---

### 4. Sequential Feed Refresh — Catastrophically Slow at Scale

- [x] Fix (2026-03-22)

**Location:** `src/core/feeds/feed-service.ts:237-238`

`refreshAllFeeds()` awaits each feed sequentially. 500 feeds x 5s = 42 minutes.

**Fix:** Use `Promise.allSettled()` with a concurrency limit (e.g., 5-10 concurrent fetches).

---

### 5. No Rate Limiting on Proxy/Sync Endpoints

- [x] Fix (2026-03-22)

**Location:** `src/core/proxy/proxy-handler.ts`, `src/core/sync/sync-handler.ts`

Unauthenticated, unlimited requests. The proxy is a DDoS amplification vector. Sync vault IDs can be brute-forced.

**Fix:** Add per-IP rate limiting middleware. Document reverse-proxy rate limiting for self-hosted users. Add response size limits to proxy handler.

---

### 6. Bug: Wrong Error Returned on Discovery Failure

- [x] Fix (2026-03-22)

**Location:** `src/core/feeds/feed-service.ts:125`

Returns `parseResult.error` when `discovery` fails. Should return `discovery.error`.

**Fix:** Trivial one-line fix. Add test to prevent regression.

## High (Design issues, impacts reliability/maintainability)

### 7. N+1 Query Pattern in exportAll()

- [x] Fix (2026-03-22)

**Location:** `src/core/storage/db.ts:363-365`

Fetches articles per-feed in a loop. 500 feeds = 501 Dexie queries during sync export.

**Fix:** Add a `getAllArticles()` bulk query or use Dexie's `toArray()` on the articles table directly.

---

### 8. Sequential IndexedDB Writes

- [x] Fix (2026-03-22)

**Location:** `db.ts:259-261` (`addArticles`), `db.ts:388-395` (`importAll`)

Encrypts and writes one-by-one. 500 feeds + 10,000 articles = 10,500 sequential operations during sync import.

**Fix:** Batch encrypt, then use Dexie `bulkPut()`.

---

### 9. ArticleItem Not Memoized

- [x] Fix (2026-03-22)

**Location:** `src/components/articles/article-item.tsx:25`

Missing `React.memo()`. With 500 articles, every parent re-render re-renders all items.

**Fix:** Wrap with `React.memo(ArticleItem)`.

---

### 10. db.ts Module-Level Mutable State

- [x] Fix (2026-03-22)

**Location:** `src/core/storage/db.ts:27-28`

`cryptoKey` and `hmacKey` are module-level `let` variables with 11 `!.` non-null assertions. No initialization guard. Any call before `open()` silently crashes.

**Fix:** Add an `isOpen()` guard or throw a descriptive error on first use if uninitialized. Consider encapsulating state in a class or closure.

---

### 11. sync-setup-dialog.tsx is 752 Lines

- [x] Refactor (2026-03-22)

**Location:** `src/components/sync/sync-setup-dialog.tsx`

16 `useState` hooks. Handles 4 views + 3 confirmation dialogs in one component.

**Fix:** Split into `SyncStatusView`, `SyncSetupWizard`, `ExistingVaultFlow`. Extract shared `ConfirmationView` helper.

---

### 12. Missing Security Headers

- [x] Fix (2026-03-22)

**Location:** `vercel.json`, `server.ts`

Both are missing: `Strict-Transport-Security`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`. Only CSP is set.

**Fix:** Add all headers to both `vercel.json` and `server.ts`. Extract to a shared constant. Add test asserting Hono and Vercel set identical headers.

---

### 13. Zero Error Observability

- [x] Fix (2026-03-22)

No structured logging, no health endpoint, no error tracking. Production debugging is blind.

**Fix:** Add privacy-respecting structured logging to stdout (JSON format). Add `/api/diagnostics` health endpoint. Add React error boundaries with localStorage error log.

## Medium (Should fix)

### 14. Passphrase Generator Modulo Bias

- [x] Fix (2026-03-22)

**Location:** `src/core/crypto/passphrase-generator.ts:8-11`

`n % 7776` on Uint32 creates ~2% entropy reduction per word. Actual entropy: ~50.8 bits instead of 51.7.

**Fix:** Use rejection sampling or switch to Uint16Array (65536 mod 7776 bias < 1%).

---

### 15. DOMPurify Config Doesn't Explicitly Restrict URI Protocols

- [x] Fix (2026-03-22)

**Location:** `src/core/parser/sanitizer.ts:3-63`

Relies on DOMPurify internals to block `javascript:` and `data:` URLs rather than explicit allowlisting.

**Fix:** Add `ALLOWED_URI_REGEXP` to explicitly allow only `http:`, `https:`, `mailto:`, and fragment URLs.

---

### 16. decodeEntities() Uses innerHTML

- [x] Fix (2026-03-22)

**Location:** `src/lib/decode-entities.ts:6`

Uses `textarea.innerHTML = str` to decode entities. Fragile, DOM-dependent, breaks in Node/worker contexts.

**Fix:** Use `DOMParser().parseFromString(str, "text/html").body.textContent` or a pure library like `html-entities`.

---

### 17. Sync Endpoint Missing nosniff Header

- [x] Fix (2026-03-22)

**Location:** `src/core/sync/sync-handler.ts:10-13`

Returns JSON without `X-Content-Type-Options: nosniff` or `Content-Security-Policy: default-src 'none'`.

**Fix:** Add security headers to the `jsonResponse` helper. Covered by item 12 if headers are centralized.

---

### 18. build-api.js Has No Error Handling

- [x] Fix (2026-03-22)

**Location:** `scripts/build-api.js`

If esbuild fails, `writeFileSync` may still run or leave temp dirs. Silent failures possible.

**Fix:** Add try/catch around esbuild. Validate bundled content has no `../src/` imports before overwriting. Only clean temp dir on success.

---

### 19. initializeReturningUser() Does Too Much

- [x] Refactor (2026-03-22)

**Location:** `src/stores/app-store.ts:48-127` — 79 lines, 5 responsibilities

Detects storage mode, loads keys, migrates legacy passphrases, restores sync credentials, validates by reading feeds, optionally pulls vault.

**Fix:** Extract into: `detectStorageMode()`, `loadOrDeriveKeys()`, `validateDatabase()`, `restoreSyncIfNeeded()`.

---

### 20. No Tests for XXE/XML Bomb Attacks

- [x] Fix (2026-03-22)

**Location:** `tests/core/parser/`

No tests for `<!ENTITY>` expansions, billion laughs, or external entity declarations. Mitigated by feedsmith and browser DOMParser, but not verified.

**Fix:** Add explicit rejection tests. If feedsmith is ever swapped, these become critical.

---

### 21. sync-store logout() Has Cross-Store Knowledge

- [x] Refactor (2026-03-22)

**Location:** `src/stores/sync-store.ts:161-169`

Uses lazy imports to reset other stores. Fragile coupling.

**Fix:** Move to `app-store` as `resetAllStores()` that calls each store's reset method.

---

### 22. HMAC Indices Leak Metadata

- [x] Document (2026-03-22)

**Location:** `src/core/storage/db.ts`

HMAC indices hide field values but reveal: number of feeds, articles per feed, growth patterns, subscription churn.

**Fix:** Document this as a known limitation in `docs/architecture.md` and `SECURITY.md`. Consider record-count padding as a future enhancement.

`"radix-ui": "^1.4.3"` is listed but all imports use specific `@radix-ui/react-*` packages.

## What's Excellent (Keep doing this)

- **Framework-agnostic core** — `src/core/` has zero React imports. Portable to any platform.
- **Result\<T\> everywhere** — No thrown exceptions for expected errors. Zero unsafe `.value` access detected across the entire codebase.
- **Contract tests** — Every client-server boundary is verified. Three-entry-point rule enforced by tests in `server.test.ts`.
- **HMAC-hashed indexes** — Queries work without exposing plaintext metadata in IndexedDB.
- **Defense-in-depth sanitization** — DOMPurify at parse time AND at render time.
- **Testing discipline** — 906 tests, 3-tier strategy (unit, structural, E2E), both desktop and mobile viewports.
- **CLAUDE.md is world-class** — 300 lines of architectural documentation that actually stays current.
- **Sync adapter abstraction** — Clean `SyncStorageAdapter` interface with 4 implementations. Model design.
- **Naming quality** — Mostly intention-revealing. `feedExistsByUrl()`, `deriveAndStoreKeys()`, `isExtractionMeaningful()`.
- **Zustand store boundaries** — Well-separated concerns, no god stores, no circular dependencies.
